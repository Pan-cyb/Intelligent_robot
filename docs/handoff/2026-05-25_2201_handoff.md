# 2026-05-25 22:01 Handoff

## Current Goal

优先优化养老陪护机器人视觉链路：保留旧 MediaPipe `person_tracker`，新增轻量 YOLO/BPU-ready 后端，第一阶段只做 person bbox + 深度定位，并保持 `/person_position`、`/person_distance`、`/fall_detected` topic 兼容。

## Important Context

- 长期规则见 `AGENTS.md`。
- 当前阶段状态见 `docs/PROJECT_STATUS.md`。
- `task_manager` 仍是整机状态机中心。
- `follower_controller` 仍只订阅兼容 topic，并只在 `/robot_mode == FOLLOWING` 时执行跟随。
- 新视觉后端不能直接控制 Nav2、底盘或整机状态。
- 坐标发布必须是 `camera_link` body frame：`x 前、y 左、z 上`。
- 本次实现是在非最终运行环境中完成的“离线/虚空”实现：当前开发环境没有 RDK X5 BPU 运行库、没有 `hobot_dnn` Python 包、没有 D-Robotics 转换后的 YOLO `.bin` 模型，也没有真实相机链路可验收。
- 因此 `bpu_yolo` 只能按 D-Robotics 常见 `hobot_dnn.pyeasy_dnn` 接口和常见 YOLOv8 单输出格式先写好代码结构；后续必须在 RDK X5 实机上用真实模型运行并按模型输出 head 微调。

## Completed This Session

- 新增 `src/person_tracker/scripts/person_tracker_bpu_node.py`：
  - `PersonDetection` / `PersonDetector` 抽象。
  - `MockPersonDetector`：默认生成画面中央 person-like bbox，用于测试 RGB/depth 订阅、深度定位和 topic 发布。
  - `BpuYoloPersonDetector`：已接 D-Robotics `hobot_dnn.pyeasy_dnn` 模型加载、BPU 推理和常见 YOLOv8 输出后处理。
  - 最大 bbox 人体选择。
  - bbox 中心、下半身、扩大窗口 median depth fallback。
  - 深度过滤：`depth_scale`、`min_depth_m`、`max_depth_m`。
  - optical frame 到 ROS body frame 转换。
  - 可选 bbox 摔倒候选确认，默认关闭。
  - `inference_every_n_frames`、`max_publish_rate_hz`、2 秒视觉摘要日志。
- `robot_server.launch.py` 新增：
  - `vision_backend`，默认 `mediapipe`。
  - `enable_bpu_vision`，默认 `false`。
  - `vision_backend:=mediapipe` 启动旧 MediaPipe 节点。
  - `vision_backend:=mock` 或 `bpu_yolo` 启动新 BPU-ready 节点。
- 新增说明文档 `src/person_tracker/doc/bpu_person_tracker.md`。
- 更新 `docs/PROJECT_STATUS.md` 记录当前视觉优化阶段。

## Known Issues

- `bpu_yolo` 后端需要在 RDK X5 上安装/启用 `hobot_dnn`，并传入 D-Robotics 转换后的 YOLO `.bin` 模型；当前开发环境无法验证这一点。
- 当前后处理支持常见 YOLOv8 单输出布局；不同 Model Zoo 模型若输出 head 不同，需要只调整 `BpuYoloPersonDetector`，不要改 ROS topic、`task_manager` 或 `follower_controller`。
- `mock` 后端默认输出中心 bbox，适合链路测试，不代表真实检测效果。
- 尚未在真实相机/RDK X5 上实测 `/person_position` hz、BPU 占用和跟随平滑度。
