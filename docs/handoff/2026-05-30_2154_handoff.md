## Known Issues

- `VAD 录音耗时` 包含等待用户开口、用户说话时长和说完后的静音确认，不等同于系统纯延迟。
- 非固定长句 TTS 仍可能较慢，尤其是闲聊长回复。
- 当前 TTS 防回声依赖 suppress/cooldown 和清空缓冲，没有做真正的声学回声消除。
- `ASR_VAD_SILENCE_MS` 过低可能截断句中停顿。

## Next Recommended Steps

1. 在目标机器 `.env` 中试调：

```bash
ASR_VAD_SILENCE_MS=700
ASR_POST_TTS_COOLDOWN_SEC=0.4
```

2. 若 TTS 后误触发，增大：

```bash
ASR_POST_TTS_COOLDOWN_SEC
```

3. 若说完后录音结束仍慢，再逐步降低：

```bash
ASR_VAD_SILENCE_MS
```

4. 若固定任务命令还慢，优先增加快速命令规则和短句 TTS 缓存，不要直接绕过 `task_manager`。
